MODEL NOTES

模型说明

这版不是简单给所有基金套同一组公式。页面会先识别基金类型，再决定该看哪些模型、用什么阈值、综合分怎么加权；你还可以自己改参数、维护观察名单、补录指数估值层。

先识别基金类型 基础模型 + 类型模型 指数估值层可手动补录 可信度单独评分 综合分按类型加权

STEP 01

先做基金分类

优先看什么

  • 先看东方财富 `FTYPE`。
  • 再结合基金名称里的 `ETF / 指数 / QDII / 债券` 等关键词。
  • 再看资产配置里的股票占比、债券占比做兜底判断。

当前分 5 类

  • 指数基金 / ETF
  • 主动权益 / 偏股混合
  • 债券基金
  • QDII / 海外权益
  • 货币基金

BASE MODELS

所有权益与债基都先跑 4 个基础模型

其中区间模型、Z-Score 模型和长周期分位模型，会用你在主页选的回看窗口，默认不是写死的 240 日。

所选窗口区间模型

区间位置 = (当前净值 - 所选窗口低点) / (所选窗口高点 - 所选窗口低点) 区间模型分数 = (1 - 区间位置) * 100

你在主页选近 1 年、近 3 年、近 5 年或全部历史，这里的区间就跟着切换。

均线偏离模型

均线偏离 = 当前净值 / 均线 - 1 单条均线得分 = clamp((带宽 - 偏离) / (2 * 带宽), 0, 1)

但这里的带宽不是固定值，会按基金类型调整。债基的带宽更窄，QDII 更宽。

Z-Score 模型

Z = (当前净值 - 所选窗口均值) / 所选窗口标准差 Z 分数 = clamp((类型阈值 - Z) / (2 * 类型阈值), 0, 1) * 100

同样会按基金类型换阈值。波动大的基金允许更宽的 Z 区间。

多周期分位模型

分位 = 历史样本中 <= 当前净值 的比例 分位模型分数 = (1 - (30% * 60日分位 + 30% * 120日分位 + 40% * 所选窗口分位)) * 100

它用 60 日、120 日和你选的主窗口一起做排序，避免只被单一极值点带偏。

TYPE MODELS

指数 / 主动权益 / QDII 的附加模型

指数真实估值层

默认先自动映射常见指数 部分常见国内指数会先自动抓官方 factsheet 的： 1. PE 2. PB 3. 股息率 再按本地估值带宽换成自动估值分 如果自动抓不到， 再由你手动录入： 1. 整体估值分位 2. PE 分位 3. PB 分位 4. 股息率分位 5. ERP 分位

这层只对指数基金 / ETF 和部分 QDII 开启。没有这层时，页面仍可看净值位置，但不等于真正看到了底层资产便宜与否。

热度 / 拥挤度模型

对 1月 / 3月 / 6月 / 1年收益率分别做反向打分 收益越热，分数越低 不同基金类型使用不同收益带宽

它不是看“涨得多就是好”，而是看这只基金是不是已经变得太热、太拥挤。

波动过滤模型

综合看： 1. 当前相对 60 日高点回撤 2. 20 日年化波动 3. 20 日波动 / 60 日波动 4. 单日估算冲击

目的是过滤“看上去低位，其实还在高波动下坠”的假便宜。

BOND MODELS

债券基金的专属模型

债基窄带回归模型

仍然看 MA20 / MA60 / MA120 但带宽缩到约 1.2% / 2.2% / 3.5%

债基波动远小于权益基金，所以需要更窄的观察带宽。

债基稳态过滤模型

综合看： 1. 20 日年化波动 2. 当前相对 60 日高点回撤 3. 债券占比和股票占比

它解决的是另一个问题：低位到底是利率回调带来的机会，还是风险暴露失控带来的下跌。

TYPE WEIGHTS

综合估值分怎么加权

指数基金 / ETF

现在不再只看净值位置，而是把基础位置、指数真实估值层、热度和波动过滤一起做加权。

基础位置约 50% 指数真实估值层约 20% 热度 / 拥挤度约 18% 波动过滤约 12%

主动权益 / 偏股混合

更重热度和波动，因为主动基金的基金经理风格会让净值位置不如指数那样直接。

基础位置约 56% 热度 / 拥挤度约 24% 波动过滤约 20%

债券基金

更重窄带回归和稳态过滤，因为债基真正重要的是“偏离够不够大”和“波动有没有失真”。

基础位置约 56% 债基窄带回归约 22% 债基稳态过滤约 22%

QDII / 海外权益

QDII 也能补一层指数估值，但权重不会像宽基指数那样高，因为还有汇率和市场结构噪音。

基础位置约 44% 指数真实估值层约 18% 热度 / 拥挤度约 19% 波动过滤约 19%

PARAMS

自定义参数会改什么

阈值

偏低阈值：默认 65 偏高阈值：默认 35

它决定综合分多少开始算“偏低”或“偏高”。你如果更保守，可以把偏低阈值继续调高。

均线带宽 / Z 系数

均线带宽系数 ↑ Z 阈值系数 ↑ = 模型更宽松

这两个系数越大，模型越不容易把轻微偏离判成低估或高估。

热度敏感度

热度敏感度 ↑ = 收益一热就更快扣分

适合那些不愿意追涨、特别在意拥挤度的人。

波动敏感度

波动敏感度 ↑ = 高波动和大冲击更快扣分

适合把“先稳住再说”放在第一位的人。

CONFIDENCE

可信度怎么算

加分项

  • 历史样本足够长。
  • 盘中估算净值可用。
  • 基金元信息完整。
  • 指数基金补录了底层指数估值分位。

减分项

  • 所选窗口样本不足，被迫退回全部历史。
  • 部分模型缺失，没有参与加权。
  • 主动基金本身只能做风格代理。
  • 指数真实估值层为空。

WORKFLOW

新增页面功能怎么用

多基金对比

一次输入多只基金，横向看综合分、可信度和指数层结果，适合做候选池筛查。

观察名单

把常看的基金放进去，后续集中刷新；只有结论明显变化时，信号记录才会新增一条。

持仓视角

录入你的持仓成本、份额和目标市值后，页面会把“便宜不便宜”改写成更接近仓位动作的话术。

导出

可以导出当前基金快照、对比区 CSV 和整个本地工作区 JSON，方便你自己留档或迁移。

READING

怎么读结果

先看类型

先确认页面把它识别成什么。如果识别错了，后面的综合分就没有意义。

再看模型是否同向

综合分只是摘要，真正重要的是基础位置模型和类型模型是不是同时偏低或同时偏高。

最后看边界

这仍然是“位置估值”，不是基本面内在价值。它能告诉你贵不贵，但不能替你判断未来一定涨跌。

货币基金例外

货币基金不做这套估值。页面会直接提示你不适用，而不是硬给一个低估/高估标签。
