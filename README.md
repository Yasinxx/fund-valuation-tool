# 基金估值页

一个零依赖、本地运行的基金估值网页。

这版不再把所有基金都按同一套规则处理。页面会先识别基金类型，再切换到对应的估值框架。

## 文件

```text
fund-valuation-tool/
  index.html
  models.html
  styles.css
  app.js
  README.md
```

## 现在能做什么

- 输入 6 位基金代码后自动抓东方财富数据
- 自动识别基金类型
- 按基金类型切换模型
- 支持 `近 1 年 / 近 3 年 / 近 5 年 / 全部历史` 窗口切换
- 同时展示每个模型自己的分数和标签
- 给一个按类型加权后的综合估值分
- 给当前结果一个单独的可信度评分
- 维护多基金对比区和观察名单
- 录入你的持仓成本 / 份额 / 目标市值
- 自定义阈值、均线带宽、热度敏感度和波动敏感度
- 导出当前基金 JSON、对比区 CSV 和整个工作区 JSON

## 当前支持的基金类型

- 指数基金 / ETF
- 主动权益 / 偏股混合
- 债券基金
- QDII / 海外权益
- 货币基金

其中货币基金会直接提示“不适用”，不会硬给一个低估或高估结果。

## 怎么用

1. 用浏览器打开 `/Users/sqz/openClaw/fund-valuation-tool/index.html`
2. 输入基金代码，例如 `161725`
3. 点“抓取并估值”
4. 先看页面识别出来的基金类型
5. 再选估值回看窗口，例如近 1 年、近 3 年、近 5 年或全部历史
6. 看各模型结果、可信度和综合结论
7. 如果是指数基金 / QDII，可以在“指数真实估值层”里补录底层指数的分位
8. 如果你常看多只基金，可以把它们加入对比区或观察名单
9. 如果你想把结果更贴近自己的纪律，就在参数区改阈值和敏感度
10. 想看公式时，打开 `/Users/sqz/openClaw/fund-valuation-tool/models.html`

如果浏览器直接从 `file://` 打开时拦了远程脚本，就在目录下起个本地静态服务：

```bash
cd /Users/sqz/openClaw/fund-valuation-tool
python3 -m http.server 8080
```

然后打开 [http://localhost:8080/](http://localhost:8080/)。

## 模型结构

### 基础模型

所有非货币基金都会先跑 4 个基础模型：

1. 所选窗口区间模型
2. 均线偏离模型
3. Z-Score 模型
4. 多周期分位模型

注意：

- 区间模型、Z-Score 模型和长周期分位模型会跟着你选的回看窗口变化
- 均线带宽和 Z-Score 阈值会按基金类型调整，不是固定常数

### 类型模型

- 指数基金 / QDII：
  - 指数真实估值层
  - 热度 / 拥挤度模型
  - 波动过滤模型
- 主动权益：
  - 热度 / 拥挤度模型
  - 波动过滤模型
- 债券基金：
  - 债基窄带回归模型
  - 债基稳态过滤模型
- 货币基金：
  - 直接提示不适用

## 怎么读

- 分数越高，表示相对当前基金类型的历史位置更偏低
- 分数越低，表示相对当前基金类型的历史位置更偏高
- 综合分只是摘要，不要只盯一个数字
- 更重要的是：基础位置模型和类型模型是否同向
- 指数基金最好补上指数估值层，否则你看到的仍然主要是净值位置，而不是底层指数贵不贵
- 可信度低时，把它当成筛查提示，不要把它当成强结论

## 新增工作流

- 对比区：一次输入多只基金做横向比较
- 观察名单：保存常看基金，集中刷新，只记录明显信号变化
- 持仓视角：录入成本、份额和目标市值后，页面会给出更贴近仓位管理的解释
- 导出：支持当前基金 JSON、对比区 CSV、工作区 JSON

## 指数真实估值层

这层主要给指数基金 / ETF 和部分 QDII 用：

- 页面会先按名称和类型尝试自动映射常见指数
- 由于公共指数估值源不稳定，这层默认采用手动录入
- 你可以补录：
  - 整体估值分位
  - PE 分位
  - PB 分位
  - 股息率分位
  - ERP 分位

另外，部分常见国内指数会优先尝试用官方 factsheet 自动补充最新 `PE / PB / 股息率`，再按本地估值带宽换算成自动估值分；如果自动抓取失败，仍然可以手动覆盖。

录入后，综合分和可信度都会一起更新

## 边界

- 这仍然是“位置估值”，不是基本面内在价值
- 它适合做筛查和辅助判断
- 它不替代行业研究、仓位纪律和风险控制
- 它不会下单，也不会生成真实交易指令
